View on GitHub Quick start

Why Refity

A lightweight, secure way to run your own Docker registry without locking you into a single cloud or storage vendor.

SFTP backend

Blobs and manifests live on your SFTP server. Use Hetzner Storage Box, any VPS, or your existing file storage.

Registry v2 API

Works with docker push and docker pull. Compatible with standard Docker CLI and tools.

Web UI

Browse groups and repos, list tags, copy pull commands, and manage access from a clean React interface.

JWT auth

Token-based authentication. Optional Hetzner usage dashboard; works without it for non-Hetzner setups.

Multi-arch

Manifest lists and multi-architecture images are supported. Tag sizes reflect real layer data.

Easy deploy

Docker Compose ready. Go backend, React frontend. SQLite for metadata; no extra databases to run.

How it works

1

Deploy Refity (backend + frontend) and point it at your SFTP server.

2

Create groups in the web UI (e.g. myteam). No auto-create: you control structure.

3

Push images with Docker: docker push your-registry/myteam/app:latest.

4

Pull from anywhere: same URL and tag. List tags and copy commands from the UI.

Quick start

git clone https://github.com/troke12/refity.git
cd refity
cp .env.example .env
# Edit .env: FTP_HOST, FTP_USERNAME, FTP_PASSWORD, etc.
docker-compose up -d

Open http://localhost:8080 for the web UI. Default login: admin / admin — change the password after first login.
